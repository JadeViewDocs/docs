# 对话框 API

**用途**：在 C 里弹出 **Windows 原生**的「选文件」「另存为」「提示框」「错误框」，不用自己画 UI。  
2.0 预览版对该模块做了重构：**同步接口默认阻塞并直接返回 JSON 字符串**，同时补充了 `*_async` 异步接口。

---

## 公共约定

| 项 | 含义 |
|----|------|
| `window_id` | 作为**父窗口**，决定对话框模态挂在谁上面。 |
| `filters` | 文件对话框里「文件类型」下拉框：值为 **JSON 数组的字符串**，元素形如 `{"name":"分组名","extensions":["jpg","png"]}`；`extensions` 里可用 `"*"` 表示所有文件。 |
| `properties` | 打开对话框行为：**逗号分隔**的特性名，如 `openFile`、`openDirectory`、`multiSelections`、`showHiddenFiles`（与 [对话框前端 API](/api-2-preview/dialog-frontend-api) 中 `properties` 数组写法对应）。 |
| 同步返回值 | `char*`（UTF-8 JSON）；使用后必须调用 `jade_text_free()` 释放。 |
| 异步回调 | `void (*callback)(const char* json_result)`；指针由库侧管理，回调返回后失效。 |

> ⚠️ **必须释放（重点）**：`jade_dialog_show_open_dialog` / `jade_dialog_show_save_dialog` / `jade_dialog_show_message_box` 的同步返回值都是 JadeView 在堆上分配的字符串。  
> 你只要拿到 `char*`，无论成功、取消还是错误返回，都应在使用完成后调用一次 `jade_text_free(ptr)`。  
> 如果不释放，长期运行会持续泄漏内存；如果重复释放或释放非 JadeView 分配的指针，可能导致崩溃。

---

## 文件对话框（统一参数结构）

2.0 预览版将旧的 `OpenDialogParams` / `SaveDialogParams` 合并为统一结构 `FileDialogParams`。

```c
typedef struct FileDialogParams {
  uint32_t window_id;
  const char *title;
  const char *default_path;
  const char *button_label;
  const char *filters;
  const char *properties;
} FileDialogParams;
```

---

## `jade_dialog_show_open_dialog`（同步）

**用途**：弹出**打开文件**对话框。该函数为同步阻塞接口，直接返回 JSON 字符串。

```c
char* jade_dialog_show_open_dialog(const FileDialogParams* params);
```

**返回 JSON 示例**：
- 成功选择：`{"canceled":false,"file_paths":["D:/a.txt"]}`
- 取消选择：`{"canceled":true,"file_paths":[]}`

---

## `jade_dialog_show_save_dialog`（同步）

**用途**：弹出**另存为**对话框。该函数为同步阻塞接口，直接返回 JSON 字符串。

```c
char* jade_dialog_show_save_dialog(const FileDialogParams* params);
```

**返回 JSON 示例**：
- 成功保存：`{"canceled":false,"file_path":"D:/out.txt"}`
- 取消保存：`{"canceled":true,"file_path":null}`

---

## `jade_dialog_show_message_box`（同步）

**用途**：弹出**带图标的提示框**（信息、警告、错误等）。该函数为同步阻塞接口，直接返回 JSON 字符串。

```c
typedef struct MessageBoxParams {
  uint32_t window_id;
  const char *title;
  const char *message;
  const char *detail;
  const char *buttons;
  int32_t default_id;
  int32_t cancel_id;
  const char *type_;
} MessageBoxParams;

char* jade_dialog_show_message_box(const MessageBoxParams* params);
```

| 字段 | 用途 |
|------|------|
| `message` | 主提示文字。 |
| `detail` | 下面小字补充说明。 |
| `buttons` | 用 `\|` 隔开多个按钮名，如 `确定\|取消`。 |
| `default_id` / `cancel_id` | 默认焦点在哪个按钮、哪个算取消。 |
| `type_` | `none`、`info`、`warning`、`error` 等，控制图标风格。 |

**返回 JSON 示例**：`{"response":1}`

---

## `jade_dialog_show_error_box`

**用途**：快速弹一个**错误提示**，标题 + 内容，一般用于严重错误、不用用户选多个按钮的场景。

```c
int32_t jade_dialog_show_error_box(uint32_t window_id, const char* title, const char* content);
```

## 异步对话框 API（新增）

```c
int32_t jade_dialog_show_open_dialog_async(
    const FileDialogParams* params,
    void (*callback)(const char* json_result));

int32_t jade_dialog_show_save_dialog_async(
    const FileDialogParams* params,
    void (*callback)(const char* json_result));

int32_t jade_dialog_show_message_box_async(
    const MessageBoxParams* params,
    void (*callback)(const char* json_result));
```

- 返回 `1/0` 表示是否成功发起。
- 用户操作完成后，通过 `callback` 返回 UTF-8 JSON 结果。
- 回调参数由库侧管理；若需长期持有，请自行拷贝字符串。

---

## Breaking Changes（接入方必须改）

- 删除 `OpenDialogParams` / `SaveDialogParams`，改用 `FileDialogParams`。
- `MessageBoxParams` 不再包含 `blocking/callback`。
- `jade_dialog_show_open_dialog` / `jade_dialog_show_save_dialog` / `jade_dialog_show_message_box` 从返回 `int32_t` 变为返回 `char*`。
- 所有同步返回的字符串都必须 `jade_text_free()` 释放。

---

## 迁移建议（最短路径）

1. 先更新 FFI 声明：结构体与 `restype/argtypes` 一并调整。
2. 同步流程改为直接读取返回 JSON，不再依赖 `blocking` 切换与回调。
3. 需要非阻塞行为时，改用 `*_async` 接口。
4. 为同步接口封装统一释放习惯：**读取结果 -> 业务处理 -> `jade_text_free`**（建议用 `try/finally` 或等价机制保证释放）。

---
