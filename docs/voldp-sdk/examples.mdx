# 示例代码

## 推荐示例工程

当前最完整、最接近真实应用组织方式的样例是 `main.wsv`。

这份示例已经覆盖：

- 启动时设置 IPC 回调
- 注册多个 IPC 通道
- 手动订阅特殊事件
- 初始化运行时并进入消息循环
- 在 `应用就绪` 中创建主窗口
- 创建托盘、添加菜单、显示通知
- 处理窗口、托盘、IPC 等事件

## 推荐工程结构

`main.wsv` 建议按下面的结构理解：

### 1. 启动方法

- 设置 IPC 频道回调
- 注册 IPC 通道
- 手动补充特殊事件订阅
- 调用 `初始化(...)`
- 进入 `运行消息循环()`

### 2. IPC 回调方法

- 统一处理所有 IPC 请求
- 按通道名分发业务逻辑
- 返回 JSON 文本结果

### 3. 应用就绪事件

- 创建窗口选项与视图选项
- 设置协议服务目录
- 创建主窗口
- 创建托盘
- 添加托盘菜单
- 显示通知

### 4. 其它事件

- 全部窗口关闭
- 托盘交互事件
- 托盘菜单命令事件
- 页面开始加载

```wsv
参数 窗口ID <类型 = 整数>
参数 数据 <类型 = 文本型>

    返回 ("")

```
