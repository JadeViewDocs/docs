# 高级用法

## 初始化与事件订阅的关系

`JadeView.初始化()` 默认会自动订阅普通事件。这样做的目的有两个：

- 简化常见场景，不必手动注册所有事件
- 避免在 `app-ready` 这类初始化阶段事件上错过时机

因此，推荐流程是：

1. 先设置 IPC 回调
2. 再注册 IPC 通道
3. 再手动补充特殊事件订阅
4. 最后调用 `初始化()`

## 自动订阅与手动订阅

### 自动订阅

默认会自动订阅大部分普通事件，例如：

- 窗口创建、关闭、状态变化
- 页面开始加载、完成加载、标题更新、图标更新
- `javascript-result`
- `postmessage-received`
- `app-ready`
- `second-instance`
- `global-hotkey`
- 通知相关事件

### 手动订阅

以下事件应显式调用 `订阅事件(...)`：

- `订阅_窗口事件.窗口即将关闭`
- `订阅_视图事件.即将导航`
- `订阅_视图事件.请求新窗口`
- `订阅_视图事件.下载开始`
- `订阅_视图事件.文件拖入`

原因：

- 前四个属于可拦截事件，是否订阅会影响默认行为
- `file-drop` 不应在所有应用中默认开启

## 可拦截事件

当前只有以下事件解释火山回调返回值：

- `window-closing`
- `webview-will-navigate`
- `webview-new-window`
- `webview-download-started`

规则：

- 返回 `1`：拦截
- 返回 `0`：继续默认行为

其它事件即使返回非零，也会被忽略。

## 非静态回调

### IPC

IPC 推荐用对象方法，而不是静态模板函数。推荐形态：

- `置IPC频道回调 (本对象, "输出名")`
- `注册IPC通道 (...)`

原因：

- IPC 本质上是面向对象的业务分发
- 回调通常需要访问对象成员、窗口状态、缓存数据
- 火山的对象方法更适合承接文本型返回值

### 异步对话框

异步对话框同样推荐对象方法回调：

- 打开文件对话框异步回调
- 保存文件对话框异步回调
- 消息框异步回调

桥接层已经负责：

- 复制底层返回的 UTF-8 JSON
- 调用 `jade_text_free(...)`
- 将结果转为火山文本后再回调对象方法

火山层不需要再关心底层指针释放。

## 第二实例事件

`second-instance` 的原始 JSON 中包含 `argv` 数组。

当前模块会：

- 解析 `argv`
- 去掉 `argv[0]` 可执行文件路径
- 将剩余参数转成火山文本数组后再送入回调

这样业务层可以直接处理真实命令行参数。

## 托盘事件

`JadeView_托盘.创建()` 会自动订阅：

- `tray-event`
- `tray-menu-command`

并且已经做了二次结构化：

- `托盘交互事件(事件类型, 托盘ID)`
- `托盘菜单命令事件(托盘ID, 键名, 项目ID, 项目类型, 已选中, 风险项目)`

使用者不需要自己再解析原始 JSON。

## 同步与异步对话框

### 同步对话框

同步接口直接返回 `文本型 JSON`。桥接层内部已经：

- 复制原始文本
- 调用 `jade_text_free(...)`
- 返回稳定火山文本

### 异步对话框

异步接口推荐用对象回调。返回值会被忽略。

## 文本编码规则

当前模块对调用者隐藏了编码细节。调用时可按火山 `文本型` 处理：

- 传入底层前由模块内部转 UTF-8
- 从底层返回后由模块内部转火山文本
