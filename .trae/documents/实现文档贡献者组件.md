# 实现文档贡献者组件

## 1. 概述

在每个文档页面下方添加文档贡献者组件，展示GitHub头像，点击后打开GitHub主页，使用Ant Design的Avatar.Group组件展示。

## 2. 实现步骤

### 2.1 安装Ant Design组件库

```bash
npm install antd
```

### 2.2 创建自定义DocItem组件

- 在 `src/theme/DocItem/` 目录下创建 `index.jsx` 文件
- 继承原有的DocItem组件，在文档内容下方添加贡献者组件

### 2.3 创建贡献者组件

- 在 `src/components/Contributors/` 目录下创建 `index.jsx` 文件
- 实现Contributors组件，使用Ant Design的Avatar.Group展示头像
- 添加点击头像打开GitHub主页的功能

### 2.4 实现贡献者数据获取

- 创建 `src/utils/getContributors.js` 文件
- 实现获取文档贡献者的逻辑
- 支持通过GitHub API或git命令获取贡献者信息

### 2.5 集成到文档页面

- 将Contributors组件添加到DocItem组件中
- 确保在每个文档下方显示

### 2.6 样式调整

- 在 `src/css/custom.css` 中添加贡献者组件的样式
- 确保与文档主题一致

## 3. 技术细节

### 3.1 贡献者数据获取

- 使用GitHub API的 `/repos/{owner}/{repo}/commits` 端点获取文件的提交历史
- 从提交历史中提取唯一的作者信息
- 缓存贡献者数据以提高性能

### 3.2 组件设计

```jsx
// Contributors.jsx
import React from 'react';
import { Avatar } from 'antd';

const Contributors = ({ contributors }) => {
  return (
    <div className="contributors-section">
      <h3 className="contributors-title">文档贡献者</h3>
      <Avatar.Group maxCount={10}>
        {contributors.map(contributor => (
          <a 
            key={contributor.login} 
            href={contributor.html_url} 
            target="_blank" 
            rel="noopener noreferrer"
          >
            <Avatar src={contributor.avatar_url} alt={contributor.login} />
          </a>
        ))}
      </Avatar.Group>
    </div>
  );
};

```

### 3.3 性能优化

- 在构建时预生成贡献者数据
- 使用缓存机制减少API调用
- 限制显示的贡献者数量

## 4. 预期效果

- 每个文档页面下方显示"文档贡献者"标题
- 展示该文档的GitHub贡献者头像
- 头像可点击，打开对应GitHub主页
- 支持浅色/深色主题切换
- 响应式设计，适配不同屏幕尺寸
