## 计划概述

创建一个新的"设计"页面，用于详细介绍JadeView的性能特性和与其他WebView框架的对比，重点突出从项目启动到窗口显示仅需16毫秒的完整流程。

## 实施步骤

1. **创建设计文档页面**
   - 在 `docs/guides/` 目录下创建 `design.mdx` 文件
   - 编写以下内容：
     - 启动性能：JadeView从项目启动 → jadeview初始化 → 创建窗口 → 显示窗口仅需16毫秒
     - 内存安全：基于Rust的内存管理机制
     - 线程安全：所有API函数的线程安全性
     - 异步架构：初始化和窗口创建的异步特性

2. **更新侧边栏配置**
   - 在 `sidebars-default.js` 中添加设计页面到侧边栏
   - 放置在API参考之后

3. **编写详细性能对比**
   - 使用表格形式展示各框架的完整启动流程性能对比
   - 分析性能差异的原因
   - 突出JadeView的优势

4. **确保内容准确和结构化**
   - 使用清晰的标题和列表组织信息
   - 提供具体的性能数据
   - 使用简洁易懂的语言

## 预期效果

- 导航栏中新增"设计"页面
- 设计页面包含完整的性能特性描述
- 清晰说明16毫秒是从项目启动到窗口显示的完整流程
- 用户可以通过表格清晰对比不同框架的性能
- 突出显示JadeView的核心优势：快速启动、内存安全、线程安全和高效的异步处理

## 关键性能数据

| 框架 | 完整启动流程时间 | 启动流程 | 架构 | 优势 |
|------|----------------|----------|------|------|
| JadeView | 16毫秒 | 项目启动 → 初始化 → 创建窗口 → 显示窗口 | Rust + wry | 极快启动，内存安全，线程安全 |
| Electron 23 | 1400毫秒 | 项目启动 → 加载Chromium → 加载Node.js → 初始化 → 创建窗口 → 显示窗口 | Chromium + Node.js | 完整生态，跨平台 |
| NW.js 0.70 | 850毫秒 | 项目启动 → 加载Chromium → 初始化 → 创建窗口 → 显示窗口 | Chromium + Node.js | 直接加载渲染进程，开发简单 |
| CEF/CefSharp | 数百毫秒 | 项目启动 → 加载Chromium → 初始化 → 创建窗口 → 显示窗口 | Chromium | 高性能，广泛应用 |
| WebView2 | 数百毫秒 | 项目启动 → 初始化WebView2 → 创建窗口 → 显示窗口 | Edge | 轻量，原生集成 |
